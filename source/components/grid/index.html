12-column Grid

The grid system requires loading helix-ui.css into your application.

Bootstrap Compatibility

  1. Avoid using Bootstrap containers
    • The Bootstrap .container class will cause an .hxRow to expand beyond what is expected.
  2. Please use the Bootstrap grid with Bootstrap form classes.
    • The .form-section, .form-control, and other form-related Bootstrap CSS classes are known to have compatibility issues with the HelixUI grid.

Explicit Columns

NOTE If the sum of a row's columns exceeds 12, the columns will wrap.
See Column Wrapping for more details.

Explicit columns will take up a defined percentage of the row.

Use the .hxCol-N class (1 ≤ N ≤ 12) to create columns that are a fraction of the row width.

  • .hxCol-B-N responsive classes are available
  • .hxCol-N is shorthand for .hxCol-xs-N

Reordering Columns

Use the .hxOrder-N classes (1 ≤ N ≤ 12) to reorder your columns.

  • .hxOrder-B-N responsive classes are available
  • .hxOrder-N is shorthand for .hxOrder-xs-N

Responsive Layout

NOTE Responsive column classes override automatic columns.

Each responsive class corresponds to the minimum breakpoint required to achieve that layout.

If you mix responsive clases, larger breakpoints will trump smaller ones.

CSS Class Minimum Breakpoint Overrides Overridden By
.hxClass-xs-N 0em (0px) N/A all below
.hxClass-sm-N 40em (~640px) all above all below
.hxClass-md-N 64em (~1024px) all above all below
.hxClass-lg-N 75em (~1200px) all above all below
.hxClass-xl-N 90em (~1440px) all above N/A

Styling the Grid

NOTE Avoid styling rows and columns.
Add a container within a column to apply styling.

Because styles for rows and columns are meticulously defined to support the grid definition, any modification to the margin, padding, or geometry of these elements will affect the layout.

  • Wrap your content in a <div> and style using a custom CSS class.
